# wfb-link v0.1.0-alpha.3

This alpha promotes the product-facing managed raw application stream path.

## Highlights

- Added `ManagedWfbStreamsBackend` for product-facing raw UDP streams.
- Supervises one WFB-NG helper process per managed stream.
- Rewrites product-facing `RawApplicationDatagram` endpoints to internal WFB
  distributor/aggregator datagram endpoints owned by `UserspaceRadioBackend`.
- Supports separate stream directions and profiles, such as video downlink,
  telemetry downlink, and sparse control uplink.
- Adds named stream health and report counters for TX/RX streams.
- Adds best-effort managed helper degradation: required helper exits fail
  readiness, while best-effort helper exits degrade only the named stream.
- Adds the receiver-backed managed-stream smoke gate:
  `scripts/run-wfb-link-managed-streams-smoke.sh`.
- Defaults that smoke to conservative stream profiles, 80 ms video pacing, and
  100 warmup payloads so repeat gates do not count WFB session acquisition or
  marginal throughput profiles as adoption failures.
- Adds repeat support in the production readiness gate via
  `MANAGED_STREAMS_SMOKE_REPEATS=N`.

## Known Limitations

- RF calibration is still not full Linux parity across all conditions. Runtime
  IQK, LCK, and EFUSE TX-power profiles remain receiver-gated.
- The Linux backend remains a contract/design stub; Linux products should use
  native WFB-NG over monitor mode for now.
- Android USB host transport is planned but not implemented.
- Helper binaries, firmware, service TOML, and key material remain product
  release artifacts.
